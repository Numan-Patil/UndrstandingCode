Bubble Sort Algorithm

A simple sorting algorithm that repeatedly steps through the list, compares adjacent elements and swaps them if they are in the wrong order.

How It Works

Bubble sort works by repeatedly comparing adjacent elements and swapping them if they're in the wrong order. The algorithm gets its name because smaller elements "bubble" to the beginning of the list.

Algorithm Steps:

  1. Step 1 Start with the first element of the array
  2. Step 2 Compare it with the next element
  3. Step 3 If the first element is greater than the second, swap them
  4. Step 4 Move to the next pair and repeat
  5. Step 5 Continue until no more swaps are needed

Pseudocode

ALGORITHM BubbleSort(arr): n = length of arr FOR i = 0 to n-1: FOR j = 0 to n-i-2: IF arr[j] > arr[j+1]: SWAP arr[j] and arr[j+1] RETURN arr

Example Walkthrough

Let's trace through sorting [64, 34, 25, 12, 22, 11, 90]:

Pass 1:

[64, 34, 25, 12, 22, 11, 90] → Compare 64 and 34, swap [34, 64, 25, 12, 22, 11, 90] → Compare 64 and 25, swap [34, 25, 64, 12, 22, 11, 90] → Compare 64 and 12, swap [34, 25, 12, 64, 22, 11, 90] → Compare 64 and 22, swap [34, 25, 12, 22, 64, 11, 90] → Compare 64 and 11, swap [34, 25, 12, 22, 11, 64, 90] → Compare 64 and 90, no swap

After pass 1, the largest element (90) is in its correct position.

Time & Space Complexity

Time Complexity:

  • Best Case: O(n) - when the array is already sorted
  • Average Case: O(n²) - typical performance
  • Worst Case: O(n²) - when array is reverse sorted

Space Complexity:

  • O(1) - only uses a constant amount of extra space

Implementation Hints

Key Points to Remember:

  • Use nested loops: outer loop for passes, inner loop for comparisons
  • Reduce the inner loop range with each pass (elements get sorted from the end)
  • Consider adding an optimization flag to detect if array is already sorted
  • Remember to swap elements when they're in wrong order

Common Mistakes:

  • Forgetting to reduce the inner loop range
  • Incorrect comparison logic (using < instead of >)
  • Not handling the swap operation correctly
  • Off-by-one errors in loop boundaries

When to Use Bubble Sort

Good for:

  • Learning and understanding sorting algorithms
  • Small datasets (less than 10 elements)
  • Nearly sorted data (with optimization)
  • Situations where simplicity is more important than efficiency

Not recommended for:

  • Large datasets
  • Performance-critical applications
  • Production systems
